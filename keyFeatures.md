# 🧠 Core Features Contract (Do Not Break)

This document defines the **non-negotiable system behavior** for the Voice Document Agent.  
All code changes, refactors, and feature additions must preserve these guarantees.

---

## 1. Document Awareness
- Agent must always be aware of:
  - all uploaded document names
  - currently selected document
- Agent must:
  - verbally list available documents when asked
  - confirm selected document before reading

---

## 2. Deterministic Document Reading (Critical 🚨)
- Document reading must **NOT be generated by LLM**
- Reading must always use:
  - stored document content
  - ordered sentence/token stream
- Agent must:
  - start reading from beginning when requested
  - continue reading until interrupted
- Reading must support:
  - pause
  - resume from exact position

---

## 3. Streaming TTS + Word-Level Sync
- TTS must be **streaming (not batch)**
- UI document reader must highlight:
  - current word being spoken
  - in real-time sync with audio
- Highlight must:
  - move forward continuously
  - match spoken output
- Resume must continue highlighting from last position

---

## 4. Interrupt Handling (Real-time)
- User interruption must:
  - immediately stop TTS
  - preserve reading position
- System must:
  - switch to Q&A mode instantly
  - not lose reading context

---

## 5. Context-Aware Q&A
- When user asks a question:
  - agent must use **recent reading context (last N sentences)**
  - not rely only on general LLM knowledge
- If insufficient context:
  - agent may use web search
- Responses must:
  - be concise
  - relate back to document when possible

---

## 6. Resume Reading After Q&A
- After answering:
  - if user says "continue"
- System must:
  - resume reading from exact previous position
  - restore word highlighting
  - continue TTS stream seamlessly

---

## 7. Notes & Snippet Saving
- When user asks to save explanation:
  - system must store:
    - explanation text
    - linked document span (word/sentence)
- Notes must:
  - be retrievable
  - persist across sessions
- Clicking linked text must:
  - open note popup

---

## 8. Highlighting System
Two distinct highlight types must exist:

| Type | Behavior |
|------|--------|
| Live Reading Highlight | moves word-by-word with TTS |
| User Highlight | static, user-defined important text |

- User highlight must:
  - persist
  - use different color than live highlight

---

## 9. Document Interaction Model (Teacher Mode)
- Agent must behave like:
  - a teacher explaining a book
- Must support:
  - explain word
  - explain sentence
  - explain concept
- Explanation must:
  - optionally include examples
  - optionally include external info

---

## 10. Export Functionality
- System must support export to:
  - PDF
  - DOCX
- Export must include:
  - original document
  - user highlights
  - saved notes
  - linked annotations

---

## 11. State Management (Critical Consistency Rule 🚨)
System must always track:

- current_document_id
- reading_position (sentence_index, word_index)
- is_reading
- notes
- highlights

State must:
- never reset on interruption
- persist during session

---

## 12. LLM Responsibility Boundaries
LLM must ONLY handle:
- answering questions
- deciding actions (read / explain / resume)

LLM must NOT:
- generate full document text for reading
- control playback timing
- manage highlighting logic

---

## 13. Continuous Session Behavior
- Session must feel:
  - conversational
  - persistent
- Agent must:
  - remember previous context within session
  - avoid restarting flow unnecessarily

---

## 14. UI Contract (Do Not Break)
- Left panel:
  - voice agent interaction
- Right panel:
  - document viewer
- Must always support:
  - real-time highlighting
  - clickable notes
  - visible document state

---

## 15. Fail-Safe Behavior
- If any component fails:
  - system must not lose:
    - reading position
    - notes
    - highlights
- System should:
  - degrade gracefully
  - allow resume without restart

---

# 🚨 Final Rule
Any change that breaks:
- reading continuity
- word-level sync
- interrupt handling
- context-aware Q&A

is considered a **critical regression and must be rejected**
